# 🚀 Projects

**Назначение:** папка для всех активных проектов. Каждый проект — отдельная папка со своими заметками, задачами, документацией. Здесь живёт текущая работа.

## Правила работы

- **Один проект = одна папка**. Называть папку кратко и понятно: `название-проекта` (kebab-case).
- Внутри папки можно создавать любую структуру, но рекомендуется:
  - `README.md` — общее описание проекта, цели, ссылки.
  - `tasks.md` — список задач (или используйте плагин Tasks).
  - `notes.md` — рабочие заметки, идеи, протоколы встреч.
  - папка `research` — исследования, прототипы, ссылки.
- Использовать теги: `#project/название` и, например, `#status/active`.
- Связывать заметки проекта с областями знаний (`Areas`) и ресурсами (`Resources`), чтобы накапливать контекст.
- После завершения проекта перемещать всю папку в `Archive`.
